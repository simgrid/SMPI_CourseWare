Topic #3 Module: LU Factorization

Introduction Activity #1 Conclusion

Overview

Objective: To implement a Gaussian elimination algorithm (without pivoting) to perform a LU factorization of a square matrix in a distributed memory manner using message passing.

1-D Data Distribution

Like the modules in the previous topic, we use a simple 1-D block data distribution. We consider a NxN matrix, A. This matrix contains single precision elements (float). The execution takes place on p processors, where p divides N. Each processor thus holds a N × N/p block column of the matrix. In other words, The process with rank 0 holds columns 0 to N/p − 1, the process with rank 1 holds columns N/p to 2N/p − 1, etc.

Roadmap

This module consists of 1 activity:

  • Activity #1: Have each MPI process allocate and initialize its own block column of matrix A, and have the processes perform the LU factorization.

What to turn in

You should turn in a single archive (or a github repo) with:

All source code
XML platform files (see details in the activities)
A Makefile that compiles all executables (and has a 'clean' target!)

Implement a program called lu, that implements the Gaussian elimination (without pivoting) algorithm. You may have covered this algorithm in a course, and it is of course well-document in various texts. Regardless, a sequential implementation, , is provided. So you can use it as a starting point in a "let's parallelize an existing sequential implementation" scenario.

You can use any XML platform file and accompanying hostfile to run your MPI program, since this module is solely about correctness and not performance. For instance, you can use cluster_crossbar_64.xml as a platform file and hostfile_64.txt as a hostfile.

For debugging purposes, remember that it's a good idea to insert calls to MPI_Barrier so that debugging output from the program can be structured step-by-step.

What have we learned?

Congratulations for making it this far. You have learned to implement a moderately complex algorithm using a 1-D data distribution . This algorithm and its implementation are very typical of parallel numerical linear algebra implementations, but it overly simplistic. In fact, many optimization techniques are known to make the parallelization more efficient, in particular to hide the communication latencies. Hiding these latencies is the theme of Topic #4.
